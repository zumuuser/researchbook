# v0.2.0 Roadmap

## Copy PDFs to App Internal Storage

**Problem:** When a user imports a PDF from Desktop (or any location), the app currently stores the original file path in the database. If the user later moves or deletes that PDF file, the app can no longer open it.

**Solution:**
- On PDF import, **copy the PDF file** into the app's internal data directory:
  - Linux: `~/.local/share/com.researchbook.app/pdfs/`
  - macOS: `~/Library/Application Support/com.researchbook.app/pdfs/`
  - Windows: `%APPDATA%\com.researchbook.app\pdfs\`
  - (Use Tauri `app_data_dir()` API for cross-platform paths)
- Store the **internal copied path** in the database, not the original user path.
- The original file can be moved/deleted by the user without breaking the app.

**Affected platforms:** All (Windows, macOS, Linux)

**Status:** Planned for v0.2.0
